# IoT & Edge Computing 파트

이 디렉토리(`iot/`)는 **ESP32 기반의 3채널 압력 센서 수집 및 펌웨어**와 **라즈베리 파이 5 엣지 게이트웨이의 지능형 상태 제어 런타임** 소스코드를 포함하고 있습니다.

하드웨어 실측 데이터 수집 계층(IoT)과 로컬 비전 AI 추론 및 분실물 처리 계층(Edge)의 유기적인 결합을 통해 도서관 사석화 및 헬스케어 인프라를 실현합니다.

<br/>

## 🛠️ 핵심 기술 스택 및 구현 로직

### 1. 3채널 FSR 매핑 및 실시간 자세 분석 (ESP32)
* **하드웨어 인터페이스**: 좌석당 3개의 FSR 406 압력 센서(좌측 엉덩이, 우측 엉덩이, 등받이)를 배치하여 아날로그 전압 값(`ADC`)을 실시간 샘플링합니다.
* **자세 분류 알고리즘**: 좌/우 압력 편차 불균형율 및 등받이 접촉 유무를 종합 연산하여 3대 대표 자세('정상', '다리 꼬기/기울어짐', '거북목/허리 숙임')를 판별하고 MQTT JSON 페이로드로 발행합니다.

### 2. YOLOv8s 십자형 사분면 매핑 및 Edge AI 추론 (Raspberry Pi 5)
* **2x2 사분면 수학적 ROI 매핑**: 단일 IP 카메라의 RTSP 비디오 스트림 중심점을 기준으로 화면을 십자(2x2) 형태로 분할하여 독립적인 4개의 좌석 구역(ROI)을 획득합니다.
* **분실물 BBox 크롭 및 S3 업로드**: YOLOv8s 커스텀 모델을 통해 10대 핵심 카테고리(배낭, 노트북, 책 등) 물품을 탐지하며, 탐지된 객체의 BBox 영역만 실시간으로 크롭하여 AWS S3에 자동 업로드하고 API 서버로 메타데이터를 전송합니다.

### 3. 딕셔너리 기반 멀티 타이머 및 융합 판정
* **병렬 공석 관리**: 4개 좌석의 실시간 센서 상태와 개별 공석 타이머를 내부 딕셔너리 구조로 매핑하여 백엔드 서버의 네트워크 부하를 분산시킵니다.
* **교차 검증 시나리오**: 부재 감지 시 백엔드 API와의 통신을 통해 실제 이용자의 '체크인/발권' 유무를 연동하여 동적 사석화 예약 타이머를 제어합니다.

## 📁 파일 구성

| 파일명 | 개발 언어 | 설명 |
| :--- | :---: | :--- |
| `add_healthcare.ino` | `C` / `C++` | 3채널 FSR 센서 데이터 정규화 및 자세 분석 알고리즘 기반 ESP32 펌웨어 코드 |
| `main_system.py` | `Python` | RTSP 스트림 파싱, YOLOv8s 추론, ROI 분할, S3 파일 업로드 및 엣지 타이머 관리 메인 프로그램 |
| `requirements.txt` | `Text` | 엣지 컴퓨팅 구동을 위한 최적화 의존성 패키지 리스트 (`ultralytics`, `opencv-python`, `paho-mqtt`, `boto3` 등) |

## 실행 가이드 (Edge Gateway 환경)

### 1. 가상환경 진입 및 필수 패키지 설치
```bash
# 가상환경 활성화 후 의존성 라이브러리 설치
$ pip install -r requirements.txt
```

### 2. 로컬 MQTT 브로커(Mosquitto) 상태 검증
엣지 게이트웨이 내부에서 ESP32 센서 노드의 이벤트를 수신하기 위해 MQTT 브로커가 정상 가동 중이어야 합니다.
*(※ 외부 디바이스 접근을 위해 `mosquitto.conf` 내에 `listener 1883` 및 `allow_anonymous true` 속성이 설정되어 있어야 합니다.)*
```bash
$ sudo systemctl status mosquitto
```

### 3. 메인 파일 실행 구성
실행 전 `main_system.py` 내부의 `CAM_URL`(카메라 RTSP 경로), AWS API 엔드포인트 세션, `AWS_ACCESS_KEY` 등 환경 설정 변수가 기입되었는지 확인 후 런타임을 구동합니다.
```bash
$ python3 main_system.py
```
